# Mapa das Informações

Pense no sistema como três blocos principais conectados entre si.

## Obras

A obra é o centro de contexto.

É nela que ficam agrupados:

- diários;
- atividades;
- efetivo;
- materiais;
- clima;
- fotos;
- anotações;
- alertas.

## Usuários

Cada usuário entra no sistema com um nível de acesso e, em geral, fica vinculado a uma obra.

Essa combinação responde duas perguntas:

- o que essa pessoa pode fazer;
- em qual obra ela pode fazer isso.

## Diários

O diário representa um dia específico dentro de uma obra.

Ele concentra o retrato operacional daquele dia e conversa com os outros módulos.

## Como tudo se conecta

- uma **obra** possui vários **usuários** e vários **diários**;
- um **diário** pertence sempre a uma **obra**;
- os registros do dia alimentam o **diário**;
- os usuários autorizados criam, revisam, aprovam ou auditam esse conteúdo.

## Regra de leitura

Se surgir dúvida sobre um dado, o caminho certo é sempre perguntar:

1. a qual obra isso pertence;
2. a qual dia isso pertence;
3. qual usuário executou ou alterou a informação.
